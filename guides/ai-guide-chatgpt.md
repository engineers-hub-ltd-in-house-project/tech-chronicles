# ChatGPT向け執筆ガイド

**OpenAI ChatGPT（GPT-4o / o1 / o3）を使った技術史連載の執筆方法**

---

## ChatGPTの特性と強み

ChatGPTは世界で最も広く使われているLLMチャットサービスであり、以下の特性が技術史連載の執筆に関連する。

- **対話的な構築に強い**: 一度に全量を渡すよりも、対話を重ねて品質を上げるアプローチが得意
- **Custom Instructions**: システムレベルの永続的な指示設定が可能
- **GPTs（カスタムGPT）**: 専用の執筆エージェントを作成・保存できる
- **Web Browsing**: リアルタイムのWeb検索・ファクトチェックが可能
- **Canvas（キャンバス）**: 長文の文書を専用エディタ上で直接編集できる
- **多様なモデル選択**: o3（推論特化）、GPT-4o（汎用）、o4-mini（軽量）を使い分け可能

---

## 推奨ワークフロー

### Step 0: Custom Instructionsを設定する

ChatGPTではCustom Instructions（カスタム指示）を設定できる。ここに著者の基本情報と文体ルールの核心を入れておく。

```
【Custom Instructions 設定例】

■ What would you like ChatGPT to know about you?

私は佐藤裕介、Engineers Hub株式会社のCEO/Technical Lead。エンジニア歴24年以上。
1990年代後半のSlackware 3.5からLinuxに入門し、CVS/SVN/Git の変遷を現場で体験してきた。
現在は技術史の連載記事を執筆中。テーマは「git ありきの世界に警鐘を鳴らす」全24回。

■ How would you like ChatGPT to respond?

・一人称は「私」。「である」調で書く
・結論から言うが、思考過程も惜しみなく見せる
・技術の歴史に敬意を払い、過去の技術を「劣った」と扱わない
・読者に問いかけを投げる。「あなたはどうだろうか」
・「〜と言われています」「一般的に〜」は使わない
・箇条書きの羅列で終わらせず、散文で語る
・「いかがでしたか？」で締めない
```

### Step 1: 指示書を分割して投入する

ChatGPTはClaudeと比較して、極めて長い指示書の細部を見落とすことがある。指示書を**論理的なブロックに分割して段階的に投入**するのが効果的。

```
【分割の目安】

メッセージ1: 著者プロフィール＋哲学（第1部）
  → 「以下は著者のプロフィールです。この人物像を記憶してください」

メッセージ2: 連載の設計思想＋トーン設計（第2部前半）
  → 「以下は連載の設計思想です」

メッセージ3: 文体サンプル＋構成テンプレート（第2部後半）
  → 「以下は文体のサンプルと各回の構成テンプレートです」

メッセージ4: 対象回の構成案（第3部から該当回）
  → 「以下は第1回の構成案です。これに基づいてアウトラインを作成してください」
```

### Step 2（推奨）: カスタムGPTを作成する

繰り返し使う場合は、カスタムGPTとして保存するのが効率的。

```
【カスタムGPT設定例】

名前: Tech Chronicles Writer - 佐藤裕介
説明: 技術史連載「git ありきの世界に警鐘を鳴らす」の執筆支援

Instructions:
（指示書の第1部〜第2部を全文貼り付け）
（「第6部：AIへの最終指示」のルールも含める）

Knowledge:
（指示書全文のファイルをアップロード）
（過去のファクトチェック結果をアップロード）
```

カスタムGPTを作成すれば、毎回の会話で指示書を再投入する必要がなくなる。

### Step 3: ファクトチェックを実行する

ChatGPTのWeb Browsing機能を使う。

```
プロンプト例：
「第1回の執筆に必要な事実を調査してください。以下の各項目について、
 公式ソースのURLを付けて回答してください。

 1. GitHubの登録開発者数（2025年の最新公式発表）
 2. Stack Overflow Developer Surveyにおけるgitの使用率
 3. diff コマンドの開発者と初出年
 ...

 注意：推測や不確実な情報は「未確認」と明示してください。」
```

**ポイント**: 「URLを付けて」と明示的に要求する。ChatGPTはソースのURLを省略しがちなので、強制する必要がある。また「未確認と明示」の指示を加えることで、架空の出典を防ぐ。

### Step 4: アウトライン→本文の2段階で書く

```
プロンプト例（アウトライン）：
「第1回のアウトラインを作成してください。
 指示書の構成テンプレートに従い、5部構成で。
 各セクションの想定文字数と扱うトピックを含めてください。」

プロンプト例（本文）：
「このアウトラインに基づいて、第1回の本文を執筆してください。
 佐藤裕介の文体で。10,000字以上を目標に。
 ファクトチェック済みのデータを正確に使用してください。」
```

### Step 5: Canvasで推敲する

ChatGPTのCanvas機能を使えば、生成された長文を専用エディタで直接編集・加筆できる。

```
プロンプト例：
「この記事をCanvasで開いてください」

Canvas上での追加指示例：
「セクション3の技術論をもう少し深掘りしてください」
「ハンズオンのコードブロックにコメントを追加してください」
```

## ChatGPT固有のテクニック

### テクニック1: o3モデルを構成設計に使う

ChatGPTのo3（推論モデル）は、論理的な構成設計に強い。連載全体の構成見直しや、各回間の論理的接続の検証に活用する。

```
プロンプト例（o3向け）：
「以下は全24回の構成案です。
 各回の『問い』が論理的に積み上がっているか検証してください。
 特に、読者が第1回から第24回まで読み進めたとき、
 『バージョン管理の本質の理解』が段階的に深まる構造になっているか。
 不自然な飛躍や重複がないか指摘してください。」
```

### テクニック2: GPT-4oを本文執筆に使う

本文の執筆はGPT-4oが適している。文体の再現、物語的な展開、読者を引き込む表現といった「文章力」が求められる工程向き。

### テクニック3: 「ロール指定」を明示する

ChatGPTは、ロール（役割）を明示すると精度が向上する傾向がある。

```
プロンプト例：
「あなたは佐藤裕介のゴーストライターです。
 佐藤の24年間の経験と哲学を完全に理解し、
 佐藤が書いたとしか思えない文章を書くことが仕事です。
 以下の指示書に基づいて執筆してください。」
```

Claudeでは「経験の定義」が有効だが、ChatGPTでは「ロール指定＋経験の定義」の組み合わせが最も効果的。

### テクニック4: Memory機能を活用する

ChatGPT Plusでは、会話を跨いだMemory機能がある。以下のような情報をMemoryに保存させると、新しい会話でも一貫性が保たれる。

```
プロンプト例：
「以下の情報を記憶してください：
 ・私は佐藤裕介。Engineers Hub株式会社のCEO。
 ・技術史連載『git ありきの世界に警鐘を鳴らす』を執筆中。
 ・一人称は『私』、である調。文体は指示書に準拠。
 ・第1回は執筆完了。第2回以降に進む。」
```

## 注意点

### 出力の長さ制限

ChatGPTは1回のレスポンスで出力できる文字数に限界がある（モデルにより異なるが、概ね日本語で5,000〜8,000字程度）。10,000字以上の記事を一度に生成するのは難しい場合がある。

**対策**: セクションごとに分割して生成する。

```
「まずセクション1（導入）とセクション2（歴史的背景）を書いてください」
→ 確認後
「続けてセクション3（技術論）を書いてください」
→ 確認後
「セクション4（ハンズオン）とセクション5（まとめ）を書いてください」
```

### 文体の「丁寧すぎる」傾向

ChatGPTは指示がない場合、「ですます調」で丁寧に書く傾向が強い。また、読者に寄り添いすぎて「いかがでしたか？」的な結びになりやすい。禁止事項リストを毎回のプロンプトにも再掲するのが安全。

### 架空のURLに注意

Web Browsing使用時でも、ChatGPTが架空のURLを生成することがある。ファクトチェック結果のURLは必ず人間が確認すること。

## 実際のプロンプト例（全体フロー）

```
--- 事前準備 ---
Custom Instructionsに著者情報と文体ルールを設定
（またはカスタムGPTを作成）

--- 会話1: ファクトチェック ---
[メッセージ1] 「第1回に必要な事実データを調査してください（項目リスト）」
[メッセージ2] 「ソースURLを再確認して、アクセス可能なものだけ残してください」

--- 会話2: 執筆 ---
[メッセージ1] 指示書の第3部から第1回の構成案を投入
[メッセージ2] 「アウトラインを作成してください」
[メッセージ3] 「承認します。セクション1-2を書いてください」
[メッセージ4] 「続けてセクション3を書いてください」
[メッセージ5] 「セクション4-5を書いてください」
[メッセージ6] 「全セクションを結合して、最終版を出力してください」
```
