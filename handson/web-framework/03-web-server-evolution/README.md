# 第3回ハンズオン：CGI vs FastCGI パフォーマンス比較

## 概要

CGIとFastCGIのパフォーマンス差を、Apache Bench（ab）を使って実測する。プロセスモデルの違いが生むパフォーマンスの差を定量的に体感する。

## 学べること

- CGIのプロセス起動コストがどれだけのオーバーヘッドになるか
- FastCGIの常駐プロセスモデルがなぜ速いのか
- Apache Bench（ab）によるWebサーバのベンチマーク手法
- プロセスの挙動観察によるボトルネック分析

## 演習一覧

| # | 演習内容                             | 確認ポイント                                   |
| - | ------------------------------------ | ---------------------------------------------- |
| 1 | CGIスクリプトの作成と動作確認        | リクエストごとにPIDが変わること                |
| 2 | FastCGIスクリプトの作成と動作確認    | PIDが同一でリクエストカウントが増加すること    |
| 3 | Apache Benchによるパフォーマンス比較 | CGI vs FastCGIのスループット差                 |
| 4 | プロセス挙動の観察                   | CGI時のプロセス増減 vs FastCGI時のプロセス安定 |

## 動作環境

- Docker（ubuntu:24.04ベースイメージ）
- Apache HTTP Server 2.4
- Perl 5
- libfcgi-perl（FCGI.pmモジュール）
- libapache2-mod-fcgid（Apache FastCGIモジュール）
- apache2-utils（abコマンド）

