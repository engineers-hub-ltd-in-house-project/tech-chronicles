# 第21回ハンズオン：CLIデザインの原則を体験する

## 概要

CLIツールの設計原則を実際に手を動かして体験する。manページの構造を読み解き、終了コードの動作を確認し、stdout/stderrの分離を実感し、三つのオプション流儀を比較する。

## 演習一覧

| 演習  | テーマ                 | 学べること                                           |
| ----- | ---------------------- | ---------------------------------------------------- |
| 演習1 | manページの構造        | NAME, SYNOPSIS, DESCRIPTIONの読み方と設計意図        |
| 演習2 | 終了コードの動作       | 0/非0の規約、sysexits.h、&&と\|\|の制御フロー        |
| 演習3 | stdout/stderrの分離    | エラーメッセージの正しい出力先とパイプラインへの影響 |
| 演習4 | オプションの三つの流儀 | UNIX伝統、GNU long option、--とダブルダッシュ        |

## 動作環境

- Docker（推奨）: `docker run -it --rm ubuntu:24.04 bash`
- または任意のLinux環境（Ubuntu 22.04以降推奨）

## 実行方法

```bash
bash setup.sh
```
