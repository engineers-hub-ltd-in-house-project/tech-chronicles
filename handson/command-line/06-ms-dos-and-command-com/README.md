# ハンズオン：MS-DOSとCOMMAND.COM――もうひとつのCLI系譜

**連載第6回対応ハンズオン**

## 概要

DOS/WindowsのCLI系譜とUNIXのCLI系譜の設計差異を実際に体験する。パイプ実装の違い（カーネルバッファ vs 一時ファイル）、ワイルドカード展開の責任（シェル vs アプリケーション）、バッチファイルとシェルスクリプトの対照比較を行う。

## 演習一覧

| 演習  | 内容                                             | 所要時間 |
| ----- | ------------------------------------------------ | -------- |
| 演習1 | DOS CLIとUNIX CLIの対照表確認                    | 5分      |
| 演習2 | パイプとリダイレクションの設計差異を実感する     | 10分     |
| 演習3 | バッチファイル vs シェルスクリプトの設計思想比較 | 10分     |

## 学べること

- DOS/Windowsコマンドの対照表（DIR/ls、COPY/cp、DEL/rm等）
- UNIXパイプ（カーネルバッファ、同時実行）とDOSパイプ（一時ファイル、逐次実行）の根本的差異
- ワイルドカード展開をシェルが行う（UNIX）かアプリケーションが行う（DOS）かの違いとトレードオフ
- DOSバッチファイル、UNIXシェルスクリプト、PowerShellの設計思想の比較
- パス区切り文字（`/` vs `\`）やオプション指定（`-` vs `/`）の歴史的背景

## 動作環境

- Docker（ubuntu:24.04イメージ）
- bash シェル

## 実行方法

```bash
# Docker上で実行
docker run --rm -it -v "$(pwd):/handson" ubuntu:24.04 bash /handson/setup.sh

# ローカルで実行（Ubuntu/Debian系）
bash setup.sh
```
