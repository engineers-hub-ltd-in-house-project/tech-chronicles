# ハンズオン：テキスト処理の系譜――ed, grep, sed, awk

**連載第8回対応ハンズオン**

## 概要

UNIXテキスト処理ツールの系譜を、edのラインエディタ操作から体験する。edのコマンド体系がgrep、sed、awkとして独立していった過程を追体験し、パイプラインの中でこれらのツールがどのように役割分担しているかを確認する。

## 演習一覧

| 演習  | 内容                                             | 所要時間 |
| ----- | ------------------------------------------------ | -------- |
| 演習1 | edを使ったテキスト編集（ラインエディタの体験）   | 10分     |
| 演習2 | grep / sed / awkのパイプライン（役割分担の確認） | 15分     |
| 演習3 | edの `g/re/p` からgrepへの変遷を追体験           | 10分     |

## 学べること

- edのラインエディタとしての操作体系（a, p, s, g, w, q コマンド）
- edの `g/re/p` コマンドがgrepとして独立した経緯
- edの `s` コマンドがsedの中核となった関係
- grep、sed、awkそれぞれの最適な用途と抽象レベルの違い
- パイプラインにおけるツールの組み合わせパターン
- awkのパターン-アクションモデルと連想配列による集計

## 動作環境

- bash シェル

docker run --rm -it -v "$(pwd):/handson" ubuntu:24.04 bash /handson/setup.sh
