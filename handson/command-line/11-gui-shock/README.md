# 第11回ハンズオン：GUIの衝撃

**GUIとCLIの認知モデルを体験する**

## 概要

この演習では、GUIとCLIの認知モデルの違いを、同一タスクの実行を通じて体験する。Ben Shneidermanの「直接操作」理論と、「再認（recognition）」vs「想起（recall）」の認知モデルを実践的に理解する。

## 学べること

- CLIが得意なタスク（一括処理、パターンマッチ、自動化）の実体験
- CLIが不得意なタスク（機能の発見、視覚的な探索）の実体験
- 「再現可能性」の概念とスクリプト化の価値
- 認知モデルの違いに基づくインターフェース選択の判断基準

## 演習一覧

| 演習 | タイトル                         | 内容                                           |
| ---- | -------------------------------- | ---------------------------------------------- |
| 1    | GUIが得意なこと、CLIが得意なこと | 100個のファイルの一括リネームでCLIの強みを体験 |
| 2    | 「再現可能性」の差を体験する     | ログ分析のスクリプト化と再利用                 |
| 3    | 「発見しやすさ」の差を体験する   | CLIの「想起」問題とaproposによる検索           |
| 4    | 認知モデルの違いを可視化する     | 同一タスクでのGUI/CLI操作手順の比較            |

## 動作環境

- Docker（ubuntu:24.04イメージ）
- bash 5.x
- GNU coreutils
- man-db（演習3で使用）

## 実行方法

```bash
# 自動セットアップスクリプトで全演習を実行
bash setup.sh

# Docker環境で実行する場合
docker run -it --rm ubuntu:24.04 bash
# コンテナ内でsetup.shの内容を実行
```
