# 第1回ハンズオン：GUI vs CLI――同じタスクで比べてみる

**対応記事**: [第1回「なぜ今、コマンドラインを語るのか」](../../../series/command-line/ja/01-why-cli-now.md)

## 概要

GUIのみで行う作業とCLIで行う作業を比較し、「組み合わせ可能性」「自動化容易性」「再現性」というCLIの構造的な強みを体感する。

## 実行方法

```bash
# セットアップスクリプトの実行
bash setup.sh
```

または記事のハンズオンセクションに記載されたコマンドを順番に実行する。

## 学べること

1. grep によるテキスト検索とパイプラインによる集計
2. シェルのforループによる一括ファイルリネーム
3. CLIの操作をスクリプト化して再現性を確保する方法
4. GUIでは困難な「操作の記録・共有・自動化」をCLIで実現する体験

## 動作環境

- Docker（ubuntu:24.04 イメージ推奨）
- または coreutils, gawk がインストールされたLinux/macOS環境
