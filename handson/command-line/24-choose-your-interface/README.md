# 第24回ハンズオン：インターフェース選定マトリクスを作成する

## 概要

コマンドライン連載の最終回ハンズオン。同じタスクを異なるインターフェース（CLI、自然言語、GUI相当の操作）で実行し、四軸フレームワーク（反復性・探索性・構造性・曖昧性）の違いを体感する。最終的に、自分の日常ワークフローのインターフェース選定マトリクスを作成する。

## 演習一覧

| # | 演習内容                                 | 学べること                                         |
| - | ---------------------------------------- | -------------------------------------------------- |
| 1 | 同じタスクを異なる方法で実行する         | 四軸フレームワークの体験                           |
| 2 | 反復タスクのスクリプト化                 | CLIの反復性に対する適合                            |
| 3 | 探索性の軸を体験する                     | CLIの発見可能性の弱点とその補完手段                |
| 4 | インターフェース選定マトリクスを作成する | 自分のワークフローを四軸で分析し最適なIFを判定する |

## 動作環境

- Docker（ubuntu:24.04 イメージ）
- 必要パッケージ: coreutils, findutils, grep, gawk, curl, jq, procps

## セットアップ

```bash
bash setup.sh
```

## 想定所要時間

約20-30分
