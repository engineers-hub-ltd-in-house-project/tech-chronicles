# 第9回ハンズオン：正規表現――CLIを支えるパターン言語

## 概要

正規表現の基礎から応用までを段階的に学ぶハンズオン。リテラルマッチから文字クラス、量指定子、グルーピングへと進み、BRE/EREの構文差異を実際に確認する。さらにバックトラッキングの指数爆発（ReDoS）を安全に体験し、正規表現エンジンの設計選択が持つ意味を理解する。

## 演習一覧

| 演習  | 内容                                     | 所要時間 |
| ----- | ---------------------------------------- | -------- |
| 演習1 | 正規表現の基礎――リテラルから文字クラスへ | 5分      |
| 演習2 | BRE vs ERE の構文差異                    | 5分      |
| 演習3 | バックトラッキングの指数爆発を体験       | 5分      |

## 学べること

- grep、sed、awkにおける正規表現の基本構文
- BRE（Basic Regular Expression）とERE（Extended Regular Expression）の差異
- 文字クラス、量指定子、アンカー、グルーピングの体系的な理解
- バックトラッキング型正規表現エンジンの危険性（ReDoS）
- 安全な正規表現パターンの書き方

## 動作環境

- Docker（ubuntu:24.04イメージ）
- bash
- grep, sed, awk（GNU版）
- python3（演習3のReDoSデモ用）

## 実行方法

```bash
# セットアップスクリプトで全演習を自動実行
bash setup.sh

# または Docker 上で直接実行
docker run --rm -it ubuntu:24.04 bash
# コンテナ内で各演習のコマンドを手動実行
```
