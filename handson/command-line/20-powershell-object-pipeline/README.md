# 第20回ハンズオン：PowerShell――テキストパイプラインへの根本的批判

## 概要

テキストパイプライン、オブジェクトパイプライン、構造化データパイプラインの三つのアプローチを実際に体験し、それぞれの設計思想の違いとトレードオフを理解する。

## 演習一覧

| 演習  | 内容                                 | 学べること                                           |
| ----- | ------------------------------------ | ---------------------------------------------------- |
| 演習1 | テキストパイプラインの強さと脆さ     | テキストパース（prayer-based parsing）の脆弱性を体験 |
| 演習2 | オブジェクトパイプラインの疑似体験   | jqによる構造化データ操作でPowerShellの設計思想を理解 |
| 演習3 | テキスト vs 構造化データの堅牢性比較 | 同一タスクでの堅牢性の違いを比較                     |
| 演習4 | 三つの設計思想の総合比較             | テキスト/オブジェクト/構造化データの使い分け         |

## 動作環境

- Docker (ubuntu:24.04)
- 必要パッケージ: procps, jq（スクリプトが自動インストール）

## 実行方法

```bash
# Dockerで実行
docker run -it --rm ubuntu:24.04 bash

# スクリプトをコンテナ内にコピーして実行
bash setup.sh
```
