# 第19回ハンズオン：モダンターミナルエミュレータの競争――GPU描画とプロトコル拡張

## 概要

ターミナルエミュレータの能力と限界を実際に確認する。
端末情報の取得、エスケープシーケンスの互換性検証、大量テキスト出力の描画性能体感、画像プロトコルの対応状況確認を通じて、ターミナルプロトコルの断片化と技術的負債を体験する。

## 演習一覧

| 演習  | 内容                         | 学べること                                         |
| ----- | ---------------------------- | -------------------------------------------------- |
| 演習1 | ターミナルの基本情報を確認   | TERM環境変数、terminfoによる端末能力の取得方法     |
| 演習2 | エスケープシーケンスの互換性 | モダンSGR属性（波線下線等）のサポート差異          |
| 演習3 | 大量テキスト出力の描画性能   | CPU描画 vs GPU描画の体感差、Unicode幅計算の負荷    |
| 演習4 | ターミナルの画像表示能力     | Sixel、kittyプロトコル、iTerm2プロトコルの対応状況 |

## 動作環境

- Docker（ubuntu:24.04ベース）
- 必要なツール: ncurses-bin
- 推奨: 複数のターミナルエミュレータで同じ演習を実行し、差異を比較する

## セットアップ

```bash
bash setup.sh
```
