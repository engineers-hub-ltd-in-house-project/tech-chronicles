# 第2回ハンズオン：コマンドライン以前の世界――パンチカードとバッチ処理

## 概要

バッチ処理の時代を疑似体験し、「対話的コンピューティング」がいかに革新的だったかを体感する。JCL風のジョブスクリプトで非対話的パイプラインを構築し、対話的処理との違いを比較する。

## 演習一覧

| 演習  | 内容                             | 学べること                             |
| ----- | -------------------------------- | -------------------------------------- |
| 演習1 | 非対話的パイプライン構築         | バッチ処理の「入力→処理→出力」モデル   |
| 演習2 | ターンアラウンドタイムの疑似体験 | バッチ処理のフィードバックループの遅さ |
| 演習3 | 対話的処理との比較               | 探索的データ分析における対話性の価値   |

## 動作環境

- Docker（推奨）: `docker run -it --rm ubuntu:24.04 bash`
- または任意のLinux/macOS環境（bash, awk, bc が利用可能であること）

## 使い方

```bash
# Docker環境の場合
docker run -it --rm ubuntu:24.04 bash
apt-get update && apt-get install -y coreutils gawk bc

# セットアップスクリプトの実行
bash setup.sh
```
