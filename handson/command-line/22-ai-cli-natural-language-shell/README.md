# 第22回ハンズオン：AI+CLIの現在を体験する

## 概要

AIが生成するCLIコマンドの正確性を検証し、CLIの知識がAI時代にも不可欠であることを体験する。findの`-mtime`オプションの挙動差異、GNU/BSD sedの非互換性、findの述語評価順序の罠、終了コードを使ったコマンド検証パターンを実践する。

## 演習一覧

| 演習  | テーマ                         | 学べること                                            |
| ----- | ------------------------------ | ----------------------------------------------------- |
| 演習1 | コマンドの正確性検証           | findの-mtime +/-/なしの違い、AIの出力を監査する観点   |
| 演習2 | 環境依存のコマンド差異         | GNU sed vs BSD sedのインプレース編集の違い            |
| 演習3 | findの評価順序の罠             | -deleteの位置による意図しないファイル削除のリスク     |
| 演習4 | 終了コードを使ったコマンド検証 | --help、--dry-run、type、終了コードによる検証パターン |

## 動作環境

- Docker（推奨）: `docker run -it --rm ubuntu:24.04 bash`
- または任意のLinux環境（Ubuntu 22.04以降推奨）

## 実行方法

```bash
bash setup.sh
```
