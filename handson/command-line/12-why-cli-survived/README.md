# 第12回ハンズオン：なぜCLIは死ななかったのか――自動化・再現性・組み合わせの力

## 概要

CLIの構造的優位性（組み合わせ可能性、再現性、スケーラビリティ）を、100個のMarkdownファイルを対象としたデータ抽出・分析タスクを通じて体験する。同じタスクをGUIで行う場合との比較を通じて、CLIが40年間生き残った理由を実感する。

## 演習一覧

| 演習  | 内容                                                       | 学べること                                        |
| ----- | ---------------------------------------------------------- | ------------------------------------------------- |
| 演習1 | 100個のMarkdownファイルからメタデータを抽出しCSVに変換     | 組み合わせ可能性（grep, sed, awk のパイプライン） |
| 演習2 | 抽出データの分析パイプライン（カテゴリ別集計、クロス集計） | テキスト処理ツールの組み合わせによる分析          |
| 演習3 | 分析スクリプトの作成と再利用                               | 再現性、共有性、バージョン管理親和性              |

## 動作環境

- Docker（ubuntu:24.04 イメージ推奨）
- bash 5.x 以上
- coreutils（grep, sed, awk, sort, uniq, wc, cut, find, head, tail）

## セットアップ

```bash
# 自動セットアップスクリプトの実行
bash setup.sh
```

または手動で Docker 環境を起動:

```bash
docker run -it --rm ubuntu:24.04 bash
```
