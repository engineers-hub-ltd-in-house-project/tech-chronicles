# 第3回ハンズオン：対話的コンピューティングの原体験

**連載「ターミナルは遺物か――コマンドラインの本質を問い直す」第3回**

## 概要

CTSSが実現した「対話的コンピューティング」の本質を、3つの演習を通じて体験する。

## 演習一覧

| 演習  | 内容                             | 学べること                                                |
| ----- | -------------------------------- | --------------------------------------------------------- |
| 演習1 | simhでUNIX V6を起動              | 1970年代の対話的シェル環境の原体験                        |
| 演習2 | タイムシェアリングシミュレーター | CPUタイムスライシングと多段フィードバックキューの動作原理 |
| 演習3 | 対話的 vs バッチ的問題解決       | フィードバックに基づく探索的思考の価値                    |

## 動作環境

- Docker（ubuntu:24.04）
- 追加パッケージ: simh, curl

## セットアップ

```bash
# Docker環境で実行
docker run -it --rm ubuntu:24.04 bash

# コンテナ内でセットアップスクリプトを実行
# （スクリプトの内容をコピーするか、マウントして実行）
bash setup.sh
```

または自動セットアップ:

```bash
bash setup.sh
```
