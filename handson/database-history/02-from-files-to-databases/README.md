# 第2回ハンズオン：CSVファイルベースのデータ管理と同時書き込みの破壊

## 概要

ファイルベースのデータ管理が抱える構造的な問題を、実際にコードを動かして体験する。

## 演習一覧

| 演習 | 内容                                                          |
| ---- | ------------------------------------------------------------- |
| 1    | データの冗長性と不整合を再現する（複数CSVファイル間の不整合） |
| 2    | ファイルロック（flock）の限界を体験する（TOCTOU問題の観察）   |

## 学べること

- 同一データが複数ファイルに分散することで生じるデータ不整合
- ファイルロック（flock）を使っても解決できない並行アクセスの問題
- TOCTOU（Time of Check to Time of Use）問題の本質
- データベースのトランザクションがこれらの問題をどう解決するか

## 動作環境

- Docker（`ubuntu:24.04`推奨）
- Python 3（Ubuntu 24.04に標準搭載）

## セットアップ

```bash
bash setup.sh
```
