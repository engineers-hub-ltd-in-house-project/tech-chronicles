# ハンズオン: SQLの宣言的性質を体験する

**データベースの地層 第5回「SQLの誕生——データベースに『言葉』が生まれた日」対応**

## 概要

SQLの宣言的性質——「何がほしいか」を宣言するだけで、「どう取得するか」はデータベースエンジンに委ねるという設計——を体験するハンズオン。EXPLAIN ANALYZEでクエリオプティマイザの判断を観察し、NULLの三値論理の罠を実際に体験し、ウィンドウ関数でSQLの表現力を実感する。

## 演習一覧

| # | 演習                         | 学べること                                   |
| - | ---------------------------- | -------------------------------------------- |
| 1 | 同じ結果を返す複数のSQL      | オプティマイザが実行計画を自動選択する仕組み |
| 2 | インデックスとオプティマイザ | インデックスの有無で実行計画が変わる理由     |
| 3 | NULLの三値論理               | NULL比較がUNKNOWNを返す挙動と NOT IN の罠    |
| 4 | ウィンドウ関数               | SQLの宣言的な表現力の進化                    |

## 動作環境

- Docker（PostgreSQL 17公式イメージを使用）
- 必要ディスク容量: 約500MB
- 所要時間: 約30分

## セットアップ

```bash
bash setup.sh
```

スクリプトがDockerコンテナを起動し、テストデータを投入した上でPostgreSQLのインタラクティブシェル（psql）に接続する。
