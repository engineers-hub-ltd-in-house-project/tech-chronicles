# 第24回ハンズオン：データベース選定の評価マトリクスを作る

## 概要

全24回の連載で学んだ知識を実践に活かすため、データベース選定の評価マトリクスを体験する。6つの判断軸（データの性質、アクセスパターン、一貫性要件、スケール要件、運用体制、エコシステム）に基づいて、複数のデータベース候補をスコアリングし、プロジェクトに最適な選択を導く。

## 演習一覧

| 演習 | 内容                                                |
| ---- | --------------------------------------------------- |
| 1    | 要件定義テンプレートの記入                          |
| 2    | ECサイトシナリオでのスコアリング                    |
| 3    | IoTシナリオでのスコアリング                         |
| 4    | アンチパターンの確認                                |
| 5    | PostgreSQLの拡張エコシステム体験（pgvector、JSONB） |

## 動作環境

- Docker（演習5のみ）
- bash
- 演習1-4はテキストベースのワークで、特別なツールは不要

## 学べること

- データベース選定を「なんとなく」から「根拠のある判断」に変える方法
- 6つの判断軸の実践的な適用手順
- PostgreSQLの拡張性が「一つのDBで複数の要件を満たす」力を持つこと
- 「選ばない」判断の重要性

## セットアップ

```bash
bash setup.sh
```
