# ハンズオン: PostgreSQLとMySQLの挙動の違いを体験する

**データベースの地層 第6回「Oracle, DB2, PostgreSQL——商用とOSSの系譜」対応**

## 概要

PostgreSQLとMySQLをDockerで同時に立ち上げ、同じスキーマ・同じクエリに対する挙動の違いを観察するハンズオン。型の厳密さ、文字列比較、外部キー制約、EXPLAIN出力の差異を通じて、各RDBMSの設計哲学の違いを体感する。

## 演習一覧

| # | 演習                   | 学べること                                       |
| - | ---------------------- | ------------------------------------------------ |
| 1 | 型の厳密さの違い       | PostgreSQLの型厳密性とMySQLの歴史的な寛容さ      |
| 2 | 文字列比較とNULLの挙動 | デフォルト照合順序の違いがクエリ結果に与える影響 |
| 3 | 外部キー制約の振る舞い | ストレージエンジンによる制約実装の差異           |
| 4 | EXPLAIN出力の読み比べ  | オプティマイザの判断の違いと出力フォーマットの差 |

## 動作環境

- Docker（PostgreSQL 17 および MySQL 8.4 公式イメージを使用）
- 必要ディスク容量: 約1GB
- 所要時間: 約30分

## セットアップ

```bash
bash setup.sh
```

スクリプトがPostgreSQLとMySQLのDockerコンテナを起動し、テストデータを投入した上で各演習を順次実行する。
