# 第1回ハンズオン：gitを使わずにバージョン管理する

**対応記事**: [第1回「なぜこの連載を書くのか」](../../../series/version-control/ja/01-why-this-series.md)

## 概要

`cp`、`diff`、`patch` の3コマンドだけで「バージョン管理もどき」を構築する。gitが内部で自動化していることを手動で体験し、バージョン管理の第一原理を理解する。

## 実行方法

```bash
# セットアップスクリプトの実行
bash setup.sh
```

または記事のハンズオンセクションに記載されたコマンドを順番に実行する。

## 学べること

1. スナップショット方式のバージョン管理（`cp -r`）
2. 差分の取得と保存（`diff -u`）
3. パッチの適用による復元（`patch`）
4. 3-way mergeとコンフリクト（`diff3`）
5. 手動管理の限界とバージョン管理ツールの必要性
