# 第24回ハンズオン：VCS評価マトリクスと技術選定の記録

## 概要

バージョン管理ツールの選定を「感覚」ではなく「構造化された判断」として行うための演習。プロジェクトの定量データ収集、コミットメッセージの品質測定、VCS評価マトリクスの作成、Architecture Decision Records（ADR）の記述を実践する。

## 演習一覧

| 演習  | 内容                                 | 学べること                                             |
| ----- | ------------------------------------ | ------------------------------------------------------ |
| 演習1 | プロジェクトの現状を定量的に把握する | リポジトリの統計データ収集と評価マトリクスへの反映方法 |
| 演習2 | コミットメッセージの品質を測定する   | 「歴史の保存」の品質がツールではなく運用に依存すること |
| 演習3 | VCS評価マトリクスを作成する          | 重み付けによる構造化された技術比較の手法               |
| 演習4 | ADRを書いてみる                      | 技術選定の理由を文書化し、再評価可能にする方法         |

## 動作環境

- Docker（ubuntu:24.04）推奨
- Git（バージョン2.x以降）
- bc（基本的な計算ツール）
- 所要時間：約25分

## 実行方法

```bash
# Docker環境で実行（推奨）
docker run -it --rm -v "$(pwd):/handson" ubuntu:24.04 bash /handson/setup.sh

# ローカルで実行
bash setup.sh
```

## 前提知識

- Gitの基本操作（commit, branch, merge, log, diff, blame）
- 第23回の「三つの本質」の理解（変更の記録、協調の仕組み、歴史の保存）
- 第1回〜第23回の内容（連載全体の知識を前提とする最終回演習）
