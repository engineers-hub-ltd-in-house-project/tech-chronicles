# 第10回ハンズオン：Subversionのブランチ・マージとGitの比較体験

## 概要

Subversionのブランチ作成・マージ操作を体験し、特にリネームを含むマージでのツリーコンフリクト問題を再現する。同じシナリオをGitで実行し、両者の操作感と設計思想の違いを確認する。

## 演習一覧

| 演習  | 内容                                       | 学べること                                      |
| ----- | ------------------------------------------ | ----------------------------------------------- |
| 演習1 | Subversionでブランチ作成                   | svn copyによるcheap copy、ブランチの基本操作    |
| 演習2 | 並行開発とマージ                           | svn:mergeinfoによるマージ追跡の仕組み           |
| 演習3 | リネームを含むマージ（ツリーコンフリクト） | copy+delete方式の限界、ツリーコンフリクトの実際 |
| 演習4 | 同じシナリオをGitで実行                    | Gitのリネーム検出、マージの自動解決             |

## 動作環境

- Docker（推奨）: `docker run -it --rm ubuntu:24.04 bash`
- 必要パッケージ: `subversion`, `git`
- 手動セットアップ: `bash setup.sh`
