# 第22回ハンズオン：Jujutsuを体験する——Gitとの違いを手で確かめる

## 概要

次世代VCSであるJujutsu（jj）をインストールし、Gitとの設計思想の違いを体験する。

## 演習一覧

| 演習 | 内容                                      | 学べること                                     |
| ---- | ----------------------------------------- | ---------------------------------------------- |
| 1    | リポジトリの初期化とGitバックエンドの確認 | JujutsuがGitリポジトリをストレージに使う仕組み |
| 2    | ステージングエリアのないワークフロー      | 自動スナップショット、describe、newの使い方    |
| 3    | オペレーションログと操作の取り消し        | 全操作の記録、undo、op log                     |
| 4    | Gitとの相互運用                           | ブックマーク、git export、Gitコマンドとの併用  |
| 5    | コンフリクトのファーストクラスサポート    | コンフリクトをコミットに記録、後から解決       |

## 動作環境

- Docker（ubuntu:24.04 推奨）
- git 2.x 以上
- curl（Jujutsuバイナリのダウンロードに使用）
- x86_64 Linux環境

## 使い方

```bash
# セットアップスクリプトで一括実行
bash setup.sh

# または Docker 環境で実行
docker run -it --rm ubuntu:24.04 bash
# コンテナ内で:
apt update && apt install -y git curl
# setup.sh の内容をコピー＆実行
```

## 注意事項

- Jujutsuは開発中のソフトウェアであり、コマンド体系が変更される可能性がある
- Jujutsuのバイナリリリースが利用できない場合、Rustツールチェーン（cargo install jj-cli）でビルドできる
- 演習5のコンフリクトマージは、jjのバージョンにより出力が異なる場合がある
