# 第14回ハンズオン：gitの最初のコミットに触れる——1,000行のコードが変えた世界

## 概要

gitの最初期のバージョン（2005年4月7日の最初のコミット e83c5163）に実際に触れ、gitの内部構造を低レベルコマンドで手動操作する。blob、tree、commitの3オブジェクトを自分の手で作成し、内容アドレッシングの仕組みを体験する。

## 演習一覧

| 演習  | 内容                                        | 学べること                                             |
| ----- | ------------------------------------------- | ------------------------------------------------------ |
| 演習1 | gitの最初のコミットを読む                   | 最初のコミットに含まれる10ファイルの構成、READMEの内容 |
| 演習2 | 低レベルコマンドでgitオブジェクトを手動作成 | blob/tree/commitの関係、plumbing vs porcelain          |
| 演習3 | 内容アドレッシングの体験                    | 同一内容=同一ハッシュの原理、.git/objectsの構造        |

## 動作環境

- Docker（推奨）: `docker run -it --rm ubuntu:24.04 bash`
- 必要パッケージ: `git`, `curl`
- 手動セットアップ: `bash setup.sh`
