# 第6回ハンズオン：CVSのブランチとマージの痛みを体感する

## 概要

第6回「ブランチとマージの悪夢——CVS時代の苦い教訓」に対応するハンズオン演習。

CVSのブランチモデルの技術的構造——RCSリビジョン番号に基づくブランチ番号体系、マージ追跡の不在、stickyタグの暗黙的状態——を実際の操作で体験する。「ブランチが怖い」という文化が生まれた技術的背景を、自分の手で確かめる。

## 学べること

- CVSのブランチ番号体系（ファイルごとに異なるブランチリビジョン番号）
- ブランチとトランクの並行開発とコンフリクト
- マージ追跡の不在による二重マージ問題
- マージ後のタグ付けによる回避策と、その運用上の脆さ
- stickyタグの仕組みと混乱

## 動作環境

- Linux（Ubuntu 24.04 推奨）
- macOS（Homebrew で `brew install cvs`）
- Windows（WSL2 推奨）

Docker を使う場合:

```bash
docker run -it --rm ubuntu:24.04 bash
apt update && apt install -y cvs
```

## 実行方法

```bash
bash setup.sh
```

| 演習 | テーマ                       | 概要                                                           |
| ---- | ---------------------------- | -------------------------------------------------------------- |
| 1    | ブランチ作成と番号体系       | ブランチリビジョン番号がファイルごとに異なることを確認         |

| 4    | 二度目のマージ               | マージ追跡不在による二重マージ問題を体験する                   |
| 5    | stickyタグの混乱             | 非ブランチタグ上でのコミット拒否と、stickyタグの解除を体験する |
