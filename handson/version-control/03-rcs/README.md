# 第3回ハンズオン：RCSで「履歴の自動記録」を体感する

## 概要

第3回「SCCS/RCS——自動化への第一歩」に対応するハンズオン演習。

RCS（Revision Control System）を実際にインストールして操作し、バージョン管理ツールによる「履歴の自動記録」を体感する。チェックイン/チェックアウト、差分表示、履歴表示、ロック機構といった基本概念が、gitの操作体系とどう対応するかを確認する。

## 学べること

- RCSのチェックイン/チェックアウト（ci/co）の基本操作
- rcsdiffによるリビジョン間の差分表示
- rlogによる変更履歴の確認
- ファイルロック方式の動作と制約
- ,v ファイルの内部構造（リバースデルタの実体）

## 動作環境

- Linux（Ubuntu 24.04 推奨）
- macOS（Homebrew で `brew install rcs`）
- Windows（WSL2 推奨）

Docker を使う場合:

```bash
docker run -it --rm ubuntu:24.04 bash
apt update && apt install -y rcs
```

## 実行方法

```bash
bash setup.sh
```

## 演習内容

| 演習 | テーマ                | 概要                                                 |
| ---- | --------------------- | ---------------------------------------------------- |
| 1    | 最初のチェックイン    | ci で初期バージョンを登録し、,v ファイルの生成を確認 |
| 2    | 編集サイクル          | co -l / 編集 / ci -u の一連の流れを体験              |
| 3    | 履歴の確認            | rlog で変更履歴を表示し、メタデータの自動記録を確認  |
| 4    | リビジョン間の差分    | rcsdiff でリビジョン間の差分を表示                   |
| 5    | ロックの衝突          | ファイルロック方式の制約を体験                       |
| 6    | ,v ファイルの内部構造 | 管理ファイルの中身を読み、リバースデルタを確認       |

## 前提知識

第1回・第2回のハンズオン（cp、diff、patch の基本操作）を完了していること。
