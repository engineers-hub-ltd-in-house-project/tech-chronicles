# 第19回ハンズオン：メーリングリスト方式のパッチ送付を体験する

## 概要

GitHubのPull Requestモデルが普及する以前から存在する、メーリングリストベースのパッチ送付ワークフローを体験する。git format-patch、git am、カバーレターの生成を通じて、Gitに組み込まれたパッチ交換の仕組みを学ぶ。

## 演習一覧

| 演習  | 内容                                       | 学べること                                           |
| ----- | ------------------------------------------ | ---------------------------------------------------- |
| 演習1 | git format-patchでパッチファイルを生成する | コミットのRFC 2822形式への変換、パッチファイルの構造 |
| 演習2 | git amでパッチを適用する                   | パッチの適用と著者情報の保持                         |
| 演習3 | カバーレターを付けたパッチセットを生成する | カバーレターの役割とLinuxカーネル開発の文化          |
| 演習4 | Pull Requestモデルとの操作フロー比較       | 両方式の利点・欠点と適用場面の理解                   |

## 動作環境

- Docker（推奨）: `ubuntu:24.04`
- 必要なツール: `git`
- 所要時間: 約15分

## 実行方法

```bash
# Docker環境（推奨）
docker run -it --rm ubuntu:24.04 bash
apt update && apt install -y git

# スクリプトの実行
bash setup.sh
```
