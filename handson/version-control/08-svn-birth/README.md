# 第8回ハンズオン：Subversionサーバの構築と基本操作

## 概要

Subversionが「CVSを正しくやり直した」ことの意味を、実際に手を動かして体験する。アトミックコミット、ディレクトリバージョニング、リネーム追跡、ブランチとタグのcheap copy、リビジョン番号の力、FSFSリポジトリの内部構造を確認する。

## 演習一覧

| 演習  | 内容                                 | 学べること                                                       |
| ----- | ------------------------------------ | ---------------------------------------------------------------- |
| 演習1 | リポジトリの作成とアトミックコミット | FSFSバックエンド、アトミックコミット、ディレクトリバージョニング |
| 演習2 | リネームの追跡                       | svn move による履歴の連続性保持                                  |
| 演習3 | cheap copy によるブランチとタグ      | svn copy、trunk/branches/tags構造                                |
| 演習4 | リビジョン番号の力                   | 連番リビジョンによる差分取得・過去参照                           |
| 演習5 | リポジトリの内部構造を覗く           | FSFSのファイル構造、リビジョンデータの形式                       |

## 動作環境

- OS: Ubuntu 24.04（Docker推奨）
- 必要パッケージ: `subversion`
- 所要時間: 約20分

## 使い方

```bash
# Docker環境（推奨）
docker run -it --rm ubuntu:24.04 bash

# セットアップスクリプトの実行
bash setup.sh
```
