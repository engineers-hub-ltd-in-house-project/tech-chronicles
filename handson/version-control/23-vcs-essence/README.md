# 第23回ハンズオン：バージョン管理の本質を「三つの問い」で再評価する

## 概要

バージョン管理の「三つの本質」——変更の記録（What changed?）、協調の仕組み（Who changed it, and how do we integrate?）、歴史の保存（Why did it change?）——を、Gitリポジトリ上で具体的に検証する演習。

## 演習一覧

| 演習  | 内容                   | 学べること                                                                  |
| ----- | ---------------------- | --------------------------------------------------------------------------- |
| 演習1 | 変更の記録を検証する   | git diffが「What」を自動計算し、コミットメッセージが「Why」を記録する仕組み |
| 演習2 | 協調の仕組みを検証する | ブランチとマージによる並行作業の統合、コンフリクトの意味                    |
| 演習3 | 歴史の保存を検証する   | git blame/git logによる変更追跡、コミットメッセージの質の重要性             |
| 演習4 | 三つの本質の自己評価   | 自分のワークフローを三つの本質で評価するワークシート                        |

## 動作環境

- Docker（ubuntu:24.04）推奨
- Git（バージョン2.x以降）
- 所要時間：約20分

## 実行方法

```bash
# Docker環境で実行（推奨）
docker run -it --rm -v "$(pwd):/handson" ubuntu:24.04 bash /handson/setup.sh

# ローカルで実行
bash setup.sh
```

- Gitの基本操作（commit, branch, merge, log, diff, blame）
- 第1回〜第22回の内容（特に第3回SCCS/RCS、第4回CVS、第8回Subversion、第15回Gitオブジェクトモデル）
