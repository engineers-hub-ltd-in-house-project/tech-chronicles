# 第14回ハンズオン：bashスクリプティングの生態系

## 概要

bashスクリプティングのベストプラクティス（`set -euo pipefail`、trap、関数化）を実践し、同時にbashスクリプトの限界（テストの困難さ、エラーハンドリングの脆弱性、型システムの不在）を体感する演習。bats-coreによるテスト、ShellCheckによる静的解析、bashとPythonの実装比較を行う。

## 演習一覧

| # | テーマ                    | 学べること                                        |
| - | ------------------------- | ------------------------------------------------- |
| 1 | set -euo pipefail の挙動  | 各オプションの動作と「無視される場面」の確認      |
| 2 | trap によるクリーンアップ | EXIT/ERRトラップの実装、POSIX標準とbash拡張の区別 |
| 3 | bats-core によるテスト    | bashスクリプトのテスティングフレームワーク体験    |
| 4 | ShellCheck による静的解析 | 典型的な警告パターンの理解と修正                  |
| 5 | bash vs Python 実装比較   | 同じタスクの実装比較による適材適所の判断          |

## 動作環境

- Docker: `ubuntu:24.04`（bash 5.2）
- 必要なパッケージ: git, bc, shellcheck, python3（演習5のPython版のみ）
- 所要時間: 約30-40分

## セットアップ

```bash
bash setup.sh
```
