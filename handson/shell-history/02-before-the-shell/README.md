# 第2回ハンズオン：バッチ処理と対話的処理を体験する

**対応記事**: [第2回「シェル以前の世界――テレタイプ、モニタプログラム、そして対話の誕生」](../../../series/shell-history/ja/02-before-the-shell.md)

## 概要

1960年代のバッチ処理的ワークフローと対話的ワークフローの違いを体感する。heredocを使った「パンチカード的」入力の再現、RUNCOMの精神に基づくコマンド組み合わせ、.rcファイルの起源確認を通じて、シェルが生まれる前の世界を追体験する。

## 実行方法

```bash
# セットアップスクリプトの実行
bash setup.sh
```

または記事のハンズオンセクションに記載されたコマンドを順番に実行する。

## 学べること

1. バッチ処理と対話的処理の根本的な違い（フィードバックループの有無）
2. heredocが「パンチカード的」な一括入力を再現する仕組み
3. RUNCOMの精神――コマンドをビルディングブロックとして組み合わせる手法
4. .rcファイルの命名規則が1963年のCTSS RUNCOMに由来すること
5. 現代のシステムに残るテレタイプ時代の痕跡（/dev/tty等）

- または bash がインストールされたLinux環境
