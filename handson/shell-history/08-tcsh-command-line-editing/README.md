# 第8回ハンズオン：tcshとコマンドライン編集――シェルがUIになった瞬間

## 概要

コマンドライン補完とコマンドライン編集の進化を体感するハンズオン。補完なし（dash）、基本補完（bash/tcsh）、高度な補完（zsh）の操作感の違いを比較し、GNU Readlineの設定とカスタム補完関数の作成を実践する。

## 演習一覧

| # | 演習内容                   | 学べること                                     |
| - | -------------------------- | ---------------------------------------------- |
| 1 | 補完なし（dash）の世界     | 補完機能がない環境でのファイル操作の不便さ     |
| 2 | tcshの補完体験             | TAB補完の基本動作、ファイル名/コマンド名補完   |
| 3 | Readlineの設定と編集モード | emacs/viモードの切り替え、inputrcの設定        |
| 4 | bash補完関数の作成         | プログラマブル補完の仕組み（complete/compgen） |
| 5 | 補完の生産性比較           | dash/bash/zshの補完機能と入力効率の差          |

## 動作環境

- Docker（ubuntu:24.04）推奨
- またはbash/tcsh/zsh/dashがインストールされたLinux環境

## セットアップ

```bash
# Docker環境の場合
docker run -it ubuntu:24.04 bash
# コンテナ内で:
bash setup.sh

# ローカル環境の場合
bash setup.sh
```
