# 第5回ハンズオン：クォーティング地獄を体験する

## 概要

Bourne shell（1979年）の言語設計における最大の罠――ワード分割とグロビング――を意図的に踏み、ダブルクォートによる抑制を体験する。さらにShellCheckによる静的解析で、クォーティング問題を自動検出する方法を学ぶ。

## 演習一覧

| 演習  | 内容                         | 学べること                                       |
| ----- | ---------------------------- | ------------------------------------------------ |
| 演習1 | ワード分割の基本メカニズム   | 未クォート変数展開によるワード分割の発生と抑制   |
| 演習2 | グロビングとワード分割の複合 | 変数内のワイルドカードがファイル名に展開される罠 |
| 演習3 | "$@" vs "$\*" の決定的な差   | 配列不在のBourne shellでの引数の安全な扱い方     |
| 演習4 | IFSの操作                    | ワード分割の区切り文字を変更した場合の挙動変化   |
| 演習5 | ShellCheckによる静的解析     | SC2086, SC2046, SC2048の検出と修正               |
| 演習6 | 安全なパターン集             | クォーティング地獄を回避する実践的テクニック     |

## 動作環境

- Docker（ubuntu:24.04）推奨
- bash または POSIX sh 互換シェルが動作するLinux環境
- ShellCheck（演習5で使用、setup.shが自動インストール）

## 使い方

```bash
# Docker環境で実行する場合
docker run -it ubuntu:24.04 bash
# コンテナ内で:
bash setup.sh

# ローカル環境で実行する場合（ShellCheckが必要）
bash setup.sh
```
