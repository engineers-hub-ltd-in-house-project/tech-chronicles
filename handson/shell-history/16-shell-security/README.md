# 第16回ハンズオン：シェルとセキュリティ――インジェクション、eval、権限昇格

## 概要

シェルスクリプトのセキュリティリスクを実際に体験し、防御手法を学ぶハンズオン。安全なDocker環境内でシェルインジェクションの基本パターンを確認し、ShellCheckによる静的解析とセキュアコーディングの原則を習得する。

## 演習一覧

| 演習  | テーマ                       | 学べること                                           |
| ----- | ---------------------------- | ---------------------------------------------------- |
| 演習1 | シェルインジェクションの基本 | クォーティングの有無がセキュリティに与える影響       |
| 演習2 | evalの危険性                 | 二重解析によるインジェクションリスクと安全な代替手段 |
| 演習3 | Shellshockの仕組み           | 環境変数関数エクスポートの脆弱性と修正内容           |
| 演習4 | ShellCheckによる監査         | 静的解析ツールでセキュリティ問題を検出・修正する方法 |

## 動作環境

- Docker（ubuntu:24.04ベースイメージ）
- bash 5.x（Ubuntu 24.04標準）
- ShellCheck（apt-getで自動インストール）

## 実行方法

```bash
# Docker環境で実行する場合
docker run -it ubuntu:24.04 /bin/bash
# コンテナ内で:
# setup.shをコピーして実行

# ローカル環境で実行する場合（Ubuntu/Debian系）
bash setup.sh
```

## 注意事項

- 演習はすべて安全な環境（Docker コンテナまたは専用ディレクトリ）で実行すること
- 本番環境でインジェクション攻撃のテストを行わないこと
- Shellshockの検証コマンドはパッチ適用済み環境では無害だが、仕組みを理解する目的で使用すること
