# 第6回ハンズオン：パイプとUNIX哲学を体験する

## 概要

Doug McIlroyが1964年に構想しKen Thompsonが1973年に実装したパイプの動作を段階的に確認し、テキストストリームの暗黙の契約の強みと限界を体験する。同じデータ処理タスクを伝統的パイプ（awk/sed/grep）とjqで実装し、可読性と堅牢性を比較する。

## 演習一覧

| 演習  | 内容                               | 学べること                                               |
| ----- | ---------------------------------- | -------------------------------------------------------- |
| 演習1 | パイプの基本動作                   | テキスト行がパイプを通じて段階的に変換される様子         |
| 演習2 | テキストパイプの限界               | JSONデータをgrep/awkで扱う困難とjqによる解決             |
| 演習3 | jqの構造化フィルタリング           | ネストされたJSONのパイプライン的フィルタ処理             |
| 演習4 | パイプのバッファと背圧             | カーネルバッファ、PIPE_BUF、SIGPIPEの仕組み              |
| 演習5 | 伝統的パイプ vs jq――同じタスク比較 | テキストログとJSONログでの集計処理の可読性・堅牢性の違い |

## 動作環境

- Docker（ubuntu:24.04）推奨
- bash または POSIX sh 互換シェルが動作するLinux環境
- jq（演習2-3, 5で使用、setup.shが自動インストール）

## 使い方

```bash
# Docker環境で実行する場合
docker run -it ubuntu:24.04 bash
# コンテナ内で:
bash setup.sh

# ローカル環境で実行する場合（jqが必要）
bash setup.sh
```
