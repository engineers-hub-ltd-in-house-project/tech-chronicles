# 第24回ハンズオン：シェル選定の評価マトリクスを作成する

## 概要

連載最終回のハンズオンとして、自分のチーム・プロジェクトに最適なシェル構成を設計するための評価マトリクスを作成する。シェルの歴史データを複数のツールで処理し、「対話用」「スクリプト用」「CI/CD用」の三層で最適なシェルを選定する。

## 学べること

- 自分のシェル環境を体系的に調査・可視化する方法
- 三層モデル（対話/スクリプト/CI・CD）に基づくシェル選定の考え方
- bash + jq と Nushell での構造化データ処理の比較
- 組織・チームレベルでのシェル標準化の判断基準

## 演習一覧

| # | 演習名                     | 内容                                                          |
| - | -------------------------- | ------------------------------------------------------------- |
| 1 | シェル構成の可視化         | 現在のシェル環境を三層で調査・表示する                        |
| 2 | 評価マトリクス作成         | チーム向けのシェル選定マトリクスをテンプレートから作成する    |
| 3 | シェル移行シミュレーション | JSONデータを複数のシェル/ツールで処理し、特性の違いを体感する |

## 動作環境

- OS: Linux（Ubuntu 22.04以降推奨）/ macOS / WSL2
- 必須: POSIX互換シェル（`/bin/sh`）
- 推奨: `jq`（JSONパーサー）
- オプション: Nushell、fish、Oils（追加課題用）

## セットアップ

```bash
# 自動セットアップ
bash setup.sh

# または手動で各演習を実行
sh exercises/01-visualize.sh
sh exercises/02-matrix.sh
sh exercises/03-simulation.sh
```
