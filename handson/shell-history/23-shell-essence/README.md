# 第23回ハンズオン：シェルの本質に立ち返る――対話・自動化・システム接点

## 概要

自分のシェル環境を「三つの軸（対話・自動化・システム接点）」で棚卸しし、再評価する。シェルの起動速度比較、bashismの検出、POSIX準拠スクリプトへの書き換えを通じて、「対話用シェル」と「スクリプト/システム用シェル」を分けて考える実践力を身につける。

## 学べること

- 自分のシェル環境（対話シェル、/bin/shの実体、利用可能なシェル）の正確な把握
- bashismの検出と、POSIX準拠スクリプトへの書き換え方法
- シェルの起動速度の違い（dash vs bash vs zsh）とその意味
- 三層モデル（対話用/スクリプト用/システム用）の実践的な適用

## 動作環境

- Docker: ubuntu:24.04 ベースイメージ（推奨）
- またはローカルのLinux/macOS環境でも実行可能
- 必要ツール: bash, dash（Ubuntu/Debianには標準搭載）

## セットアップ

```bash
# Dockerコンテナの起動（推奨）
docker run -it --rm ubuntu:24.04 bash

# セットアップスクリプトの実行（コンテナ内で）
bash setup.sh
```

ローカル環境で実行する場合:

```bash
bash setup.sh
```
