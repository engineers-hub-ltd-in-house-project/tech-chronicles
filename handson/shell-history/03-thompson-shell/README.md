# 第3回ハンズオン：Thompson shellの世界を体験する

## 概要

UNIX V1（1971年）に搭載されたThompson shellの制約と設計思想を、現代のシェル環境で再現・体験する演習集。パイプなし・変数なし・制御構造が外部コマンドという「何もないシェル」の世界を体感することで、現代のシェルが積み重ねてきた機能の意味を理解する。

## 演習一覧

| 演習  | 内容                                             |
| ----- | ------------------------------------------------ |
| 演習1 | リダイレクトだけでデータ処理を行う（パイプなし） |
| 演習2 | 「変数なし」の世界を体験する                     |
| 演習3 | if/gotoが外部コマンドだった世界を再現する        |
| 演習4 | fork/execモデルを観察する                        |
| 演習5 | Thompson shellの機能一覧を実際に確認する         |

## 学べること

- パイプの不在がデータ処理に与える影響（中間ファイルの必要性）
- 名前付き変数がないことによるスクリプトの保守性への影響
- 制御構造を外部コマンドとして実装する設計の意味と制約
- fork/execモデルによるプロセス生成の仕組み
- Thompson shellの機能と現代のシェルの機能の対比

## 動作環境

- Docker環境（ubuntu:24.04）またはbashが使えるLinux環境
- 特別なパッケージのインストールは不要

## 使い方

```bash
# Docker環境で実行する場合
docker run -it ubuntu:24.04 bash
# コンテナ内で setup.sh を実行

# ローカル環境で実行する場合
bash setup.sh
```
