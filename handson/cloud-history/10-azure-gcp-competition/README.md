# 第10回ハンズオン：Azure、GCP——寡占と競争の構造

## 概要

AWS、GCP、AzureのCLI（コマンドラインインターフェース）を実際にインストール・操作し、各社の設計思想の違いを体感する。CLIの構造はプラットフォーム全体の設計思想を最も直接的に反映するため、コマンドの構文と出力制御を比較するだけで三社の哲学の違いが見えてくる。

## 学べること

- 三社のCLIコマンド構造の設計思想の違い（APIラッパー vs リソース中心 vs フラット構造）
- 仮想マシン操作コマンドの構文比較
- 出力フォーマット制御の違い（JMESPath vs gcloud --format vs az --output）
- リソース管理モデルの違い（タグ vs プロジェクト vs リソースグループ）

## 動作環境

- Docker（Ubuntu 24.04ベースコンテナ）
- AWS CLI v2
- Google Cloud SDK（gcloud）
- Azure CLI（az）
