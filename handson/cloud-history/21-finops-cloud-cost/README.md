# ハンズオン：FinOps——クラウドコストという新しい工学

**連載「クラウドの考古学」第21回 対応ハンズオン**

## 概要

Infracost を使って Terraform コードからデプロイ前にインフラコストを見積もり、設計変更がコストに与える影響を定量的に確認する。FinOps の「Shift Left」——設計段階でコストを可視化する——を体験するハンズオンである。

## 学べること

- Infracost CLI のセットアップと基本的な使い方
- Terraform コードからのコスト見積もり（`infracost breakdown`）
- アーキテクチャ変更前後のコスト比較（`infracost diff`）
- EC2ベース構成とサーバーレス構成のコスト比較
- ライトサイジング、NAT Gateway 削除、S3 ライフサイクルポリシーの効果
- タグ戦略の設計とタグなしリソースの検出方法

## 動作環境

- OS: Linux（Ubuntu 24.04 推奨）/ macOS
- Docker: 利用可能であれば `ubuntu:24.04` イメージで実行可能
- 必須ツール:
  - curl
  - unzip
  - Terraform CLI（1.0 以上）
  - Infracost CLI（セットアップスクリプトが自動インストール）
- オプション:
  - AWS CLI（演習3のタグ検出で使用。AWS アカウントが必要）
  - Infracost API キー（無料で取得可能: `infracost auth login`）

## セットアップ

```bash
./setup.sh
```

## 注意事項

- Infracost はクラウドプロバイダーの公開料金データを使って見積もりを行う。実際の請求額とは異なる場合がある
- 演習1・2 は Terraform の `plan` を生成するだけで、実際のリソースはデプロイしない
- 演習3 の AWS CLI コマンドは、AWS アカウントへの認証が必要
- 見積もり結果はリージョン（東京: ap-northeast-1）の料金に基づく
