# 第7回ハンズオン：Xen、KVM——オープンソース仮想化が切り拓いた道

## 概要

KVMとlibvirtを使って仮想マシンのライフサイクル管理を体験する。virshコマンドによる「宣言的なVM管理」はクラウドAPIの原型であり、virtioによるI/O最適化の設計思想を理解する。

## 学べること

- KVM環境のセットアップとハードウェア仮想化支援（VT-x/AMD-V）の確認方法
- libvirt/virshによる仮想マシンの宣言的定義とライフサイクル管理
- virshコマンドとクラウドAPI（AWS CLI等）の設計上の対応関係
- virtioデバイスとエミュレーションデバイスの設計思想の違い

## 動作環境

- Docker（Ubuntu 24.04ベース）
- `--privileged` オプションが必要
- ホストマシンがKVM（Intel VT-x または AMD-V）をサポートしていることが望ましい
  - KVMが利用できない環境では一部の演習が制限される
