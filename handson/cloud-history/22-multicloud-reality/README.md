# ハンズオン：マルチクラウドの現実を体験する

**クラウドの考古学 第22回「マルチクラウドの現実——理想と実務のギャップ」対応**

## 概要

Terraform を使って AWS と GCP に同じ「Web サーバー + データベース」構成を定義し、マルチクラウドの抽象化の限界を体験する。

## 学べること

- Terraform で同じインフラを異なるクラウドに定義する方法
- AWS と GCP のリソースモデルの構造的差異（VPC、サブネット、ファイアウォール、マネージドDB）
- 「言語の統一」と「概念の統一」の違い
- マルチクラウド判断マトリクスを使った意思決定フレームワーク

## 演習一覧

| # | 演習内容                                     | 所要時間 |
| - | -------------------------------------------- | -------- |
| 1 | AWS / GCP の Terraform 構成を比較する        | 20分     |
| 2 | AWS vs GCP の差異を分析する                  | 15分     |
| 3 | マルチクラウド判断マトリクスを作成・評価する | 15分     |

## 動作環境

- OS: Ubuntu 24.04（Docker 推奨）
- 必須ツール: Terraform >= 1.5
- クラウドアカウント: なし（コード構造の分析が主目的。実際のデプロイは任意）
- エディタ: HCL のシンタックスハイライト対応のもの推奨
