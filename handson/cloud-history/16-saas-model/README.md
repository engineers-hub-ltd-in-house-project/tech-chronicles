# 第16回ハンズオン：マルチテナントSaaSの基本アーキテクチャ

## 概要

SaaSバックエンドの核心であるテナント分離戦略を、PostgreSQLとDockerを使って実際に構築・比較する。3つの主要なテナント分離モデル（プールモデル、ブリッジモデル、サイロモデル）を実装し、それぞれの特性とトレードオフを体感する。

## 学べること

- PostgreSQLのRow-Level Security（RLS）によるプールモデルのテナント分離
- スキーマ分離（Schema-per-Tenant）によるブリッジモデルの構築
- 3つのテナント分離戦略の比較と使い分けの判断基準
- マルチテナント環境でのデータアクセス制御の設計パターン

## 演習一覧

| 演習 | テーマ                         | 内容                                                                |
| ---- | ------------------------------ | ------------------------------------------------------------------- |
| 1    | プールモデル（RLS）            | PostgreSQL Row-Level Securityによるテナント分離の実装と動作確認     |
| 2    | ブリッジモデル（スキーマ分離） | Schema-per-Tenantモデルの構築とsearch_pathによるテナント切り替え    |
| 3    | 比較分析                       | 3つのモデル（サイロ・ブリッジ・プール）の特性比較とユースケース判断 |

## 動作環境

- Docker（PostgreSQL 16コンテナを使用）
- Bash（Linux / macOS / WSL2）
