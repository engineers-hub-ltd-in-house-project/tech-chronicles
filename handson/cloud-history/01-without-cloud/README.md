# 第1回ハンズオン：クラウドなしでサーバを立てられるか

## 概要

QEMUを使って仮想マシンを手動で構築し、クラウドが隠蔽している抽象化のレイヤーを体感する演習。

## 学べること

- 仮想ディスクの作成とフォーマット（EBSボリュームの原型）
- CPU・メモリの割り当て（インスタンスタイプの原型）
- 仮想ネットワークの構築（VPCの原型）
- QEMUのオプションとEC2パラメータの対応関係

## 演習一覧

| # | 演習内容                   | 対応するクラウド概念             |
| - | -------------------------- | -------------------------------- |
| 1 | QEMUで仮想マシンを起動する | EC2インスタンスの起動            |
| 2 | リソース制限を変更する     | インスタンスタイプの変更         |
| 3 | 仮想ネットワークを構築する | VPC / ENI / セキュリティグループ |

## 動作環境

- Docker Desktop または Docker Engine
- ベースイメージ: `ubuntu:24.04`
- 必要ディスク容量: 約500MB
- 推奨メモリ: 2GB以上

## セットアップ

```bash
# 自動セットアップ
bash setup.sh

# または手動でDockerコンテナを起動
docker run -it --rm --privileged ubuntu:24.04 bash
```

## 注意事項

- `--privileged` フラグが必要（KVM/TAPデバイスの操作のため）
- QEMUはエミュレーションモードで動作するため、実行速度は遅い。これは意図的な設計であり、クラウドの高速さとの対比を体感するためのものである
- Alpine LinuxのISOは演習中にダウンロードする（約60MB）
