# クラウドの考古学 第4回 ハンズオン

## IPMI/BMCの概念とサーバのリモート管理を体感する

### 概要

コロケーション時代の運用課題——物理サーバのリモート管理——を、BMCシミュレータを通じて体験する。IPMI/BMCの「OSとは独立した管理チャネル」という設計思想を理解し、データセンターの電力冗長化設計の意味を定量的に確認する。

### 演習一覧

| 演習  | 内容                         | 学べること                       |
| ----- | ---------------------------- | -------------------------------- |
| 演習1 | BMCシミュレータの構築と操作  | OSとは独立した管理チャネルの概念 |
| 演習2 | 深夜のディスク障害シナリオ   | リモートからの障害判断と対応判断 |
| 演習3 | 電力冗長化のシミュレーション | N/N+1/2N構成の信頼性の違い       |

### 動作環境

- Docker（Ubuntu 24.04ベースコンテナ）
- Python 3（標準ライブラリのみ使用）
- Linux標準ツール

### 実行方法

```bash
# Docker環境で実行
docker run -it --rm --name colo-handson ubuntu:24.04 bash

# セットアップスクリプトを取得して実行
# （リポジトリをクローン済みの場合）
bash setup.sh
```
