# 第8回ハンズオン：AWS EC2（2006年）——「サーバを借りる」概念が変わった日

## 概要

AWS CLIを使ってEC2インスタンスのライフサイクル管理を体験する。UserDataスクリプトによる初期設定自動化を学び、「使い捨てサーバ」の設計思想を理解する。

## 学べること

- AWS CLI v2のセットアップとEC2操作の基本コマンド体系
- EC2インスタンスのライフサイクル（run → stop → start → terminate）
- virshコマンド（第7回）とAWS CLIの設計上の対応関係
- UserDataスクリプトによるサーバ初期設定の自動化
- 「ペット」モデルと「家畜」モデルの設計思想の違い

## 動作環境

- Docker（Ubuntu 24.04ベース）
- AWS CLI v2（スクリプト内で自動インストール）
- LocalStackは不要（CLIの構造理解が目的）
