# 第24回ハンズオン：インフラ選定フレームワーク——70年の知識を意思決定に活かす

## 概要

メインフレームからサーバーレスまでの全24回で扱ったインフラ選択肢を俯瞰し、自分のプロジェクトに最適なインフラアーキテクチャを選定するための評価マトリクスと意思決定フレームワークを構築する。「なぜそのインフラを選ぶのか」を言語化する力を養う。

## 学べること

- ワークロード特性（トラフィック変動性、成長予測、レイテンシ要件、データ主権、チーム能力）の定量的な分析手法
- 計算資源の3原則（抽象化・弾力性・共有）に基づくインフラ評価の考え方
- 各インフラ選択肢（ベアメタル、VPS、IaaS、PaaS、コンテナ/K8s、サーバーレス、エッジ）の特性比較
- 意思決定フローに基づく推奨インフラの導出
- インフラ選定根拠の言語化とドキュメント化
