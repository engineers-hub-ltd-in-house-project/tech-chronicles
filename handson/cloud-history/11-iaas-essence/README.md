# 第11回ハンズオン：VPCをゼロから手動で構築する

## 概要

IaaS（Infrastructure as a Service）のネットワーク抽象化を、VPC（Virtual Private Cloud）の手動構築を通じて体感する。マネジメントコンソールのウィザードやIaCツールを使わず、AWS CLIで一つずつリソースを作成することで、VPCの構造を「部品」のレベルで理解する。

## 学べること

- VPCの基本構造（VPC、サブネット、ルートテーブル、インターネットゲートウェイ）
- パブリックサブネットとプライベートサブネットの違いがルーティングで決まること
- セキュリティグループの「ステートフルファイアウォール」としての動作
- IaaSのネットワーク抽象化を構成する個々のリソースとその関係性
- クリーンアップ時の依存関係（削除順序の重要性）

## 演習一覧

| 演習 | 内容                                     | 作成するリソース                  |
| ---- | ---------------------------------------- | --------------------------------- |
| 1    | VPCの骨格を構築する                      | VPC                               |
| 2    | サブネットを作成する                     | パブリック/プライベートサブネット |
| 3    | インターネットゲートウェイとルーティング | IGW、ルートテーブル               |
| 4    | セキュリティグループの設定               | セキュリティグループ              |
| 5    | 構築したVPCの確認と解体                  | （クリーンアップ）                |

## 動作環境

- Docker（Ubuntu 24.04）
- AWS CLI v2
- AWSアカウント（無料枠の範囲で実行可能、VPC自体は課金対象外）

## セットアップ

```bash
bash setup.sh
```

## 注意事項

- AWSアカウントとアクセスキーが必要
- VPC、サブネット、IGW、ルートテーブル、セキュリティグループ自体は課金対象外だが、EC2インスタンスやNATゲートウェイを作成した場合は課金が発生する
- ハンズオン終了後は必ずクリーンアップを実行すること
- セキュリティグループの `0.0.0.0/0` 設定はハンズオン用。実運用では自分のIPに限定すること
