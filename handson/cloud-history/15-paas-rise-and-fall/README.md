# 第15回ハンズオン：PaaSの栄枯盛衰——なぜ「便利すぎる抽象化」は苦戦したか

## 概要

第一世代PaaS（Herokuモデル）とPaaS 2.0（コンテナベース）の設計思想の違いを手元の環境で体験する。PaaSのロックイン構造、コンテナ標準化がもたらしたポータビリティの変化を定量的に分析する。

## 学べること

- 第一世代PaaSの制約（Procfile、Buildpack、揮発性FS、環境変数設定）
- PaaS 2.0（Dockerfileベース）との設計上の違い
- PaaSプラットフォーム間のロックイン度の比較分析
- The Twelve-Factor Appの原則がPaaSの設計にどう埋め込まれているか

## 演習一覧

| 演習 | 内容                                           |
| ---- | ---------------------------------------------- |
| 1    | 第一世代PaaSの制約をシミュレートする           |
| 2    | PaaS 2.0（コンテナベース）との対比             |
| 3    | PaaSプラットフォームのロックイン度を測定・分析 |

## 動作環境

- Docker（Ubuntu 24.04ベースコンテナ）
- Python 3.x
- 追加の外部サービスアカウントは不要

## 実行方法

```bash
# Docker環境で実行
docker run -it --rm ubuntu:24.04 bash

# セットアップスクリプトを取得して実行
# （リポジトリをクローン済みの場合）
bash setup.sh
```
