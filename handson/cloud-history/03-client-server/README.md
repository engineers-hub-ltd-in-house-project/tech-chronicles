# ハンズオン：ソケット通信でクライアント/サーバモデルを体感する

## 概要

Berkeleyソケット（1983年、BSD 4.2）を使って簡易的なクライアント/サーバアプリケーションを構築し、「計算を分散する」ことの意味を体感する。最小のTCP通信、RPCの原型実装、複数クライアントの同時接続処理を通じて、クライアント/サーバモデルの設計原則と限界を理解する。

## 演習一覧

| 演習  | 内容                          | 学べること                             |
| ----- | ----------------------------- | -------------------------------------- |
| 演習1 | 最小のクライアント/サーバ通信 | ソケット通信の基本パターン             |
| 演習2 | 計算の分散（RPCの原型）       | ローカル呼び出しとリモート呼び出しの差 |
| 演習3 | 複数クライアントの同時接続    | 並行処理モデルとスケーラビリティの課題 |

## 動作環境

- Docker（Docker Desktop または Docker Engine）
- ベースイメージ: `ubuntu:24.04`
- 使用言語: Python 3（標準ライブラリのみ）

## セットアップ

```bash
# 自動セットアップスクリプトの実行
bash setup.sh
```

または手動で以下を実行:

```bash
docker run -it --rm --name cs-handson ubuntu:24.04 bash
apt-get update && apt-get install -y python3 net-tools iproute2 procps
```
