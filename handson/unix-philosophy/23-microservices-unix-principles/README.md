# 第23回ハンズオン：マイクロサービスとUNIX原則――思想の転生

## 概要

UNIXパイプラインとマイクロサービスの構造的アナロジーを、同じデータ処理タスクを二つのアプローチで実装することで体感する。さらに、分散システム固有の困難（ネットワーク遅延、部分障害）を模擬し、UNIXのローカルパイプラインとの決定的な違いを理解する。

## 演習一覧

| 演習 | 内容                             | 学べること                                     |
| ---- | -------------------------------- | ---------------------------------------------- |
| 1    | UNIXパイプラインによるデータ処理 | パイプラインの合成可能性と各コマンドの単一責務 |
| 2    | マイクロサービス的なプロセス分割 | JSONベースのサービス間通信とサービス境界の設計 |
| 3    | 分散システムの困難の模擬         | レイテンシ、部分障害、リトライの必要性         |

## 動作環境

- Docker（ubuntu:24.04イメージ）
- 必要パッケージ: curl, jq, coreutils, netcat-openbsd, procps

## 実行方法

```bash
bash setup.sh
```
