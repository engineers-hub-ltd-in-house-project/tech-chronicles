# ハンズオン：「"Everything is a file"——抽象化の極致」

**UNIXという思想 第6回 対応ハンズオン**

## 概要

UNIXの「すべてはファイルである」という設計原則を、実際に手を動かして体験する。/procファイルシステムの探索、デバイスファイルの動作確認、ファイルディスクリプタの統一性の検証、FUSEによる自作ファイルシステムの実装を通じて、この抽象化の威力と限界を理解する。

## 演習一覧

| 演習  | 内容                             | 学べること                                   |
| ----- | -------------------------------- | -------------------------------------------- |
| 演習1 | /procからプロセスの内部を覗く    | 仮想ファイルシステムによるカーネル情報の公開 |
| 演習2 | デバイスファイルの動作確認       | /dev/null, /dev/zero, /dev/urandomの仕組み   |
| 演習3 | Cプログラムでfd統一性を検証      | 異なるファイルタイプに同じread()が使える     |
| 演習4 | straceでVFS層を観察              | システムコールレベルでの統一性の確認         |
| 演習5 | PythonでFUSEファイルシステム実装 | ユーザ空間でのファイルシステム構築           |

## 動作環境

- Docker（ubuntu:24.04イメージ）
- `--privileged`フラグが必要（デバイスファイルとカーネルパラメータへのアクセスに使用）
- 必要パッケージ: gcc, python3, fuse3, libfuse3-dev, pkg-config, strace

## セットアップ

```bash
bash setup.sh
```
