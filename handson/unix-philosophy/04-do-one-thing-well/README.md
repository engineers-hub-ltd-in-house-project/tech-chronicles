# 第4回ハンズオン：「"Do one thing and do it well"——単一責務の起源」

## 概要

UNIXの「一つのことをうまくやれ」という設計原則を、実際のデータ処理タスクを通じて体感する。同じ処理をUNIXコマンドのパイプラインとPythonスクリプトの2通りで実装し、設計思想の違いを比較する。

## 演習一覧

| 演習  | 内容                                               |
| ----- | -------------------------------------------------- |
| 演習1 | UNIXパイプラインによるアクセスログ分析             |
| 演習2 | Pythonスクリプトによる同じ分析                     |
| 演習3 | 設計思想の比較（性能・柔軟性）                     |
| 演習4 | パイプラインの応用——複数の分析を素早く組み合わせる |

## 学べること

- UNIXコマンド群（awk, sort, uniq, cut, head, grep）の組み合わせによるデータ処理
- パイプラインの各段階を分解して理解する方法
- 単機能コマンドの組み合わせとモノリシックスクリプトの設計思想の差異
- 同一のコマンド群で異なる分析を素早く試行する技法

## 動作環境

- Docker（ubuntu:24.04イメージ）
- Python 3（コンテナ内でインストール）

## セットアップ

```bash
bash setup.sh
```

または手動で:

```bash
docker run -it --rm ubuntu:24.04 bash
apt-get update && apt-get install -y python3
```
