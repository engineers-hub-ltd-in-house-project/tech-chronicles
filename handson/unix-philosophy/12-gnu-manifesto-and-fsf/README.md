# 第12回ハンズオン：GNU宣言とFSF——自由ソフトウェアという思想

## 概要

GNUプロジェクトがLinuxディストリビューションの基盤としてどれだけ広範に使われているかを確認し、BusyBoxの代替実装との比較を通じてGNUの設計判断を理解する。さらに、GPLライセンスの実態とソースコードへのアクセスを体験する。

## 演習一覧

| # | 演習内容                 | 学べること                                  |
| - | ------------------------ | ------------------------------------------- |
| 1 | GNU coreutilsの特定      | 日常コマンドの大半がGNU由来であることの確認 |
| 2 | GNUコンポーネントの確認  | GCC、Bash、glibc等のGNU基盤の全体像         |
| 3 | BusyBoxとの比較          | GNU coreutilsとBusyBoxの設計思想の違い      |
| 4 | GPLライセンスの確認      | 実際のライセンス条文の読み方                |
| 5 | ソースコードへのアクセス | GPLが保証するソースコード取得の実践         |

## 動作環境

- Docker Desktop または Docker Engine
- ベースイメージ: `ubuntu:24.04`（GNU環境）、`alpine:3.20`（BusyBox比較用）
- 特権モード不要

## セットアップ

```bash
./setup.sh
```
