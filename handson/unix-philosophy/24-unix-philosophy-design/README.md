# 第24回ハンズオン：UNIX哲学による設計レビュー

## 概要

最終回のハンズオンでは、UNIX哲学の5つの原則を自分のプロジェクトに適用する設計レビューを行う。技術を操作するのではなく、思考を操作する演習だ。

## 演習一覧

| 演習  | 内容                           | 学べること                 |
| ----- | ------------------------------ | -------------------------- |
| 演習1 | コンポーネントの責務分析       | 単一責務原則の適用         |
| 演習2 | インタフェースの統一性チェック | 統一インタフェースの重要性 |
| 演習3 | 設計レビューチェックリスト     | UNIX哲学の実践的適用       |

## 動作環境

- Docker（ubuntu:24.04推奨）
- または任意のUNIX系OS（macOS、Linux）
- 必要なツール：bash、jq、tree

## セットアップ

```bash
bash setup.sh
```

## 学べること

- モノリシックなコンポーネントの責務分析方法
- インタフェースの統一性を検証する技法
- UNIX哲学に基づく設計レビューチェックリスト（8項目）の活用

## 前提知識

- 基本的なシェル操作（cd、cat、grep）
- jqコマンドの基礎（JSONパーサ）
- ソフトウェア設計の基本概念（責務、インタフェース、結合度）
