# ハンズオン：ディストリビューションの差異を体感する

**UNIXという思想 第14回「ディストリビューション戦争——多様性というUNIXの遺伝子」対応**

## 概要

Docker上で複数のLinuxディストリビューション（Alpine、Debian、Fedora、Arch）を並べ、同じタスクを実行してパッケージ管理と設計思想の違いを体感する。

## 学べること

- 各ディストリビューションの基本構成の違い（Cライブラリ、コアユーティリティ、シェル）
- パッケージマネージャの操作と設計思想の比較（apk、apt、dnf、pacman）
- 依存関係管理の仕組みと可視化
- パッケージファイルの内部構造（.deb、.rpm）
- コンテナイメージサイズとディストリビューション設計の関係

## 動作環境

- Docker 20.10 以上
- インターネット接続（パッケージリポジトリへのアクセスに必要）
- ディスク空き容量: 約2GB（各ディストリビューションのイメージ合計）

## セットアップ

```bash
# 自動セットアップスクリプトを実行
bash setup.sh
```

または手動で各イメージを取得:

```bash
docker pull alpine:3.21
docker pull debian:bookworm-slim
docker pull fedora:41
docker pull archlinux:latest
```
