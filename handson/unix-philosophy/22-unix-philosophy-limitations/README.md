# 第22回ハンズオン：UNIX哲学の限界を体験する

## 概要

UNIXパイプラインが「苦手とする」タスクに挑み、UNIX哲学の構造的な限界を肌で感じるハンズオン。型なしテキストストリームの脆弱性、状態管理の困難さ、エラーハンドリングの貧弱さ、セキュリティモデルの粗さを実際に体験する。

## 演習一覧

| 演習  | テーマ                           | 内容                                                           |
| ----- | -------------------------------- | -------------------------------------------------------------- |
| 演習1 | 型なしテキストストリームの脆弱性 | ファイル名のスペースや特殊文字でパイプラインが壊れる様子を確認 |
| 演習2 | 状態を持つ処理の困難さ           | JSONデータのJOIN処理をシェルパイプラインで試み、非効率さを体感 |
| 演習3 | エラーハンドリングの限界         | パイプラインの終了コードの握りつぶしとPIPESTATUSの限界を確認   |
| 演習4 | セキュリティモデルの粗さ         | uid/gidモデルの粒度の粗さとsetuidの危険性を確認                |

## 学べること

- テキストパースに依存するパイプラインがいかに脆弱か
- 構造化データの結合処理がパイプラインでいかに不自然か
- 終了コードだけでは「何のエラーか」を伝えられない
- uid/gidモデルでは「最小権限の原則」を実現しにくい
- これらの限界が、PowerShell、Nushell等の新しいアプローチを生んだ背景

## 動作環境

- Docker（`ubuntu:24.04`イメージ）
- 必要パッケージ: `jq`, `curl`, `coreutils`

## セットアップ

```bash
bash setup.sh
```

`setup.sh` を実行すると、Docker環境内で全演習が自動実行される。
