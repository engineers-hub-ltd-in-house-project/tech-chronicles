# 第16回ハンズオン：Linuxカーネル開発の現場を覗く

## 概要

Linuxカーネルのソースツリーを探索し、コミットログを読み、MAINTAINERSファイルの構造を確認することで、世界最大のオープンソースプロジェクトの統治構造と開発規模を体感する。カーネルモジュールのコード構造やパッチ形式についても確認し、カーネル開発の「作法」を理解する。

## 演習一覧

| 演習  | 内容                                   | 学べること                                           |
| ----- | -------------------------------------- | ---------------------------------------------------- |
| 演習1 | カーネルソースツリーの構造を理解する   | ディレクトリ構造、コード規模、サブシステムの分離     |
| 演習2 | MAINTAINERSファイルを読む              | メンテナ階層、ステータス分類、サブシステム統治       |
| 演習3 | コミットログからカーネル開発の規模体感 | 貢献者の所属組織、コミット頻度、サブシステム別活動量 |
| 演習4 | カーネルモジュールのコード構造を読む   | MODULE_LICENSE、init/exit、stable-api-nonsense       |
| 演習5 | パッチの形式を理解する                 | git format-patch、Signed-off-by、DCO                 |

## 動作環境

- Docker が利用可能な環境
- 使用イメージ: `ubuntu:24.04`
- インターネット接続（カーネルソースのcloneのため）
- ディスク空き容量: 約2GB（shallow clone）

## セットアップ

```bash
bash setup.sh
```

## 注意事項

- カーネルソースのgit cloneにはネットワーク帯域と時間が必要（shallow cloneで最小化）
- 演習はカーネルのソースコードを読むのみで、コンパイルやカーネルモジュールのロードは行わない
