# UNIXという思想

**パイプ、プロセス、ファイル――すべてはここから始まった**

著：佐藤裕介（Engineers Hub株式会社 CEO / Technical Lead）

---

## 連載概要

UNIXの思想は技術ではなく設計哲学である。その原則を知らずにソフトウェアを設計している者は、車輪を再発明し続ける。

Docker、Kubernetes、マイクロサービス——モダンな技術スタックの裏側で、1969年にKen ThompsonとDennis RitchieがPDP-7の前で練り上げた設計哲学が今も生きている。「小さなプログラムを組み合わせて大きな仕事をする」「すべてはファイルである」「テキストを共通のインタフェースにする」——これらの原則は、半世紀以上にわたってソフトウェアの世界を静かに支配し続けてきた。

この連載は、UNIXの設計哲学の誕生から現代への系譜を辿り直し、24年間UNIXと歩んだエンジニアが自身の体験とともに語る全24回。UNIXを使えとは言わない。UNIXの設計哲学を「知って」使え——それがこの連載の提案である。
